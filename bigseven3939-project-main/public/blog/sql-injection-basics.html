SQL Injection cơ bản cho người mới

2026-05-20 · Web Security · SQLi

SQL Injection (SQLi) là lỗ hổng cho phép kẻ tấn công chèn câu lệnh SQL độc hại vào input của ứng dụng, từ đó đọc/sửa dữ liệu trái phép trong cơ sở dữ liệu.

1. Ví dụ kinh điển

Giả sử ứng dụng xây dựng truy vấn bằng cách nối chuỗi:

SELECT * FROM users
WHERE username = '$user' AND password = '$pass';

Nếu nhập ' OR '1'='1 vào ô mật khẩu, điều kiện luôn đúng và kẻ tấn công có thể đăng nhập mà không cần mật khẩu.

2. Cách phòng chống

  • Dùng prepared statements / parameterized queries.
  • Validate và sanitize mọi input từ người dùng.
  • Áp dụng nguyên tắc đặc quyền tối thiểu cho tài khoản DB.
  • Dùng WAF như một lớp phòng thủ bổ sung.

SQL Injection Basics for Beginners

2026-05-20 · Web Security · SQLi

SQL Injection (SQLi) is a vulnerability that lets an attacker inject malicious SQL statements through application input, allowing unauthorized reading or modification of database data.

1. The classic example

Suppose the application builds a query via string concatenation:

SELECT * FROM users
WHERE username = '$user' AND password = '$pass';

Entering ' OR '1'='1 in the password field makes the condition always true, letting an attacker log in without a valid password.

2. Prevention

  • Use prepared statements / parameterized queries.
  • Validate and sanitize all user input.
  • Apply least privilege to database accounts.
  • Use a WAF as an additional defense layer.

3. Where to practice?

Practice safely and legally on: PortSwigger Web Security Academy, TryHackMe, DVWA.

⚠️ Only practice in authorized lab environments. Attacking real systems without permission is illegal.
